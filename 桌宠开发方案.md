一、核心架构
━━━━━━━━━━━━━━━━━━━━
  👁️ 视: llava (图片理解) + ScreenCapture (屏幕) + Camera (外界)
  👂 听: Vosk (语音识别) + 语音唤醒 (Porcupine)
  🗣️ 说: FishSpeech (音色克隆) + sherpa-onnx VITS (本地备用)
  🧠 大脑: AstrBot API (DeepSeek) + ActionEngine (动作规划)
  🖐️ 动手: pyautogui (虚拟手) + 精细化操作 (邮件内容解析等)
  🔄 进化: 自适应技能学习 (自我进化)
  ⏰ 主动: 环境感知 + 定时汇报 + 智能提醒

  🟢 P2 (高级智能)
  ─────────────────
  【🔄 自我进化能力 — 完整设计】
  
  ① 技能注册系统 (✅ 已实现)
  □ 技能表: 名称/触发关键词/Python代码/版本号/优先级/别名/创建时间
  □ 原子技能库: 预置文件操作/网络请求/文本处理等基础技能
  □ 冲突管理: 关键词重叠时询问用户(覆盖/共存/取消)
  
  ② 技能生成器 (✅ 已实现)
  □ 新任务→LLM生成Python脚本→用户确认→注册
  □ 安全检查: 静态扫描危险API + 路径白名单
  □ 执行引擎: 子进程沙盒隔离
  □ 错误自愈: 经验库 + LLM修复(上限3次)
  
  ③ Agent自身能力进化 (新增，待实现) ← ★ 重点
  □ 错误日志系统: 记录每次代码修改引入的bug类型
    - 漏导入、漏连接、UI重叠等常见错误模式
    - 每次修复后自动记录: [错误类型] → [修复方案]
  □ 修改前自检清单 (强制执行):
    1. 语法检查 (py_compile)
    2. 引用检查 (被修改的文件是否被其他地方引用)
    3. UI检查 (按钮是否连接了函数)
    4. 回滚方案 (如果改坏了如何恢复)
  □ 测试驱动: 每次修改前先写测试用例, 通过后才算修复完成
  □ 经验积累: 错误→修复对积累到一定量后, 自动优化开发流程
  
  ④ 进阶功能 (保留，待实现):
  □ 安全沙盒升级: RestrictedPython或Docker容器
  □ 技能链式调用: 技能A调用技能B(skill_registry.call)
  □ 代码相似度检测: 避免重复技能
  □ 伪代码预览: 生成前先让用户确认计划
  □ 技能导出/导入: JSON格式跨设备同步
  □ 基于用户反馈的强化微调: 点赞/踩收集
